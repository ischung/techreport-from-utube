# TechReport from YouTube PRD

**작성일**: 2026-04-23 | **버전**: v0.1 | **상태**: Draft | **작성자**: Insang Cho (insang@hansung.ac.kr)

## 1. 프로젝트 개요

### 1.1 Core Goal

> **교수·강사의 영상 구조화 부담을 제거하여, 1편당 강의용 기술보고서 확보 시간을 '시간 단위'에서 '분 단위'로 단축한다.**

### 1.2 배경 및 목적

- **대상 사용자 (WHO)**: 교육 콘텐츠를 준비하는 교수 / 강사
- **사용자 Pain Point (PAIN)**: YouTube 영상의 원문 자막·전사는 두서 없는 텍스트라, 강의·리포트에 바로 쓸 수 있는 형태가 아니다. 구조화에 시간이 두 배로 든다.
- **기대 효과 (VALUE)**
  - **주**: 이해·번역·구조화의 인지적 부담을 AI가 대신 수행 → 매주 강의 준비의 심리적·시간적 압박 경감
  - **보조**: 재사용 가능한 산출물(마크다운 기술보고서) 확보 → 강의 노트·학생 자료·블로그 글에 즉시 붙여넣기 가능

### 1.3 범위 (Scope)

**In-Scope (v1 포함)**

1. 키워드 입력
2. 최근 1개월 이내 YouTube 영상 **5개** 검색 및 표시 (제목 / 링크 / 업로드 날짜)
3. 영상 1개 선택
4. 선택 영상의 내용 분석 (자막·전사 기반)
5. 한국어 기술보고서(마크다운) 생성 및 파일 저장

**Out-of-Scope (v1 제외)**

| 기능 | 제외 사유 (YAGNI) |
|------|-----------------|
| 영상 썸네일 이미지 표시 | 제목·링크만으로 선택 충분, 핵심 흐름 외 |
| 보고서 히스토리 / 저장 기록 관리 | OS 파일 시스템이 이미 해결 |
| 사용자 계정 / 로그인 | 개인 로컬 도구로 시작 |
| 여러 영상 동시 분석 / 비교 보고서 | v1은 1편에 집중해 완성도 확보 |
| 보고서 템플릿 커스터마이징 | 표준 템플릿 1개로 먼저 검증 |
| 검색 기간·채널 필터 조정 | 고정값(최근 1개월)로 시작, 피드백 후 확장 |
| 보고서 공유 / 퍼블리싱 | 마크다운 파일 복사로 대체 가능 |
| 별도 영어→한국어 번역 모드 | 분석 과정에 한국어 재구성이 포함됨 |

## 2. 대상 사용자

**Primary Persona**: **김교수 (가칭), 45세, 수도권 대학 소프트웨어공학과 교수**

- 주 3~5회 강의를 직접 준비하며, 매 학기 강의 자료에 최신 AI / LLM / SW 공학 동향을 반영하고자 한다.
- 영어 영상을 이해하는 데는 문제 없지만, 2시간짜리 발표·키노트를 일일이 시청·요약할 시간이 없다.
- 결과물을 "강의 슬라이드·학생 배포 자료·개인 블로그"의 세 채널에 재사용한다.

**핵심 니즈**
- **기능적**: 최근 1개월 내 핵심 영상을 짧은 입력으로 즉시 찾고, 구조화된 한국어 문서로 받는다
- **감성적**: "매주 강의 준비 시간에 쫓기지 않는" 여유

## 5. 상세 기능 요건

### 5.1 키워드 입력

- **Logic**
  1. 사용자가 한국어 또는 영어 키워드를 한 줄로 입력한다
  2. 시스템이 입력값을 공백 trim 등 기본 정규화한다
  3. 검색 기능(5.2)을 호출한다
- **Validation**
  - 최소 2자 이상, 최대 100자
  - 공백만 입력은 거부
- **Edge Case**
  - 데이터 없음(빈 입력): "검색할 키워드를 입력해주세요."
  - 오류 발생: "키워드를 인식하지 못했어요. 다시 입력해주세요."

### 5.2 최근 1개월 영상 검색 및 표시

- **Logic**
  1. 키워드 + '최근 1개월' 조건으로 YouTube에서 검색한다
  2. 업로드 날짜 내림차순으로 상위 5개를 선별한다
  3. 각 항목의 **제목 / 링크(URL) / 업로드 날짜**를 목록으로 표시한다
- **Validation**
  - 반드시 1개월 이내 영상만 포함
  - 최대 5개
- **Edge Case**
  - 데이터 없음: "최근 1개월 이내 관련 영상을 찾지 못했어요. 키워드를 바꿔서 다시 시도해보세요."
  - 오류 발생: "검색 중 문제가 생겼어요. 잠시 후 다시 시도해주세요." + 재시도 옵션

### 5.3 영상 선택

- **Logic**
  1. 표시된 5개 중 하나를 번호(1~5) 또는 클릭으로 선택한다
  2. 선택 영상의 메타데이터(제목, 링크, 업로드일)가 분석 단계로 전달된다
- **Validation**
  - 1~5 범위 외 입력은 거부
  - 한 번에 하나만 선택
- **Edge Case**
  - 선택 없이 진행 시도: "영상을 먼저 선택해주세요."
  - 오류: "선택을 처리하지 못했어요. 다시 시도해주세요."

### 5.4 영상 내용 분석

- **Logic**
  1. 선택 영상의 자막 또는 전사 텍스트를 확보한다
  2. 핵심 개념·주요 예시·논리 흐름·인용 가능한 수치/키워드를 추출·정리한다
  3. 강의 자료로 쓸 수 있는 한국어 구조(개요 / 본문 / 핵심 요약 / 참고)로 재조직한다
- **Validation**
  - 자막·전사 확보에 실패하면 즉시 중단 (빈약한 보고서 생성 방지)
- **Edge Case**
  - 데이터 없음 (자막 없음): "해당 영상은 자막을 제공하지 않아 분석할 수 없어요. 다른 영상을 선택해주세요."
  - 오류 발생: "분석 중 오류가 발생했어요. 다시 시도해주세요."

### 5.5 기술보고서 생성 및 저장

- **Logic**
  1. 분석 결과를 마크다운 형식으로 렌더링한다 (섹션: 개요 / 핵심 개념 / 상세 내용 / 강의 활용 팁 / 참고 링크)
  2. 파일명 규칙: `YYYY-MM-DD-[영상제목-슬러그].md`
  3. 현재 작업 디렉토리에 저장
  4. 저장 완료 후 파일 경로를 사용자에게 안내
- **Validation**
  - 파일명 공백 → 하이픈으로 자동 변환
  - 동일 파일명 존재 시 덮어쓰기 확인
- **Edge Case**
  - 쓰기 권한 없음: "저장할 수 없어요. 다른 폴더를 지정해주세요."
  - 오류: "보고서 저장에 실패했어요. 재시도해주세요."

### 5.6 인터랙션 흐름

| 사용자 행동 | 시스템 반응 |
|------------|------------|
| 키워드 입력 | 최근 1개월 영상 5개 목록 표시 (제목·링크·날짜) |
| 영상 번호 선택 | 진행 상태 표시 (스피너 + "자막 가져오는 중…", "한국어로 구조화하는 중…") |
| 분석 완료 | 마크다운 보고서 파일 생성 후 저장 경로 안내 |
| 오류 발생 | 친근·명확한 메시지 + 재시도 또는 이전 단계 복귀 옵션 |

## 6. UI/UX 요건

- **레이아웃**: **단계형 (Wizard)** — ① 키워드 입력 → ② 영상 목록 → ③ 선택 → ④ 분석 진행 → ⑤ 저장 완료
- **로딩 상태**: **스피너 + 현재 단계를 설명하는 짧은 상태 메시지** (예: "자막 가져오는 중…", "한국어로 구조화하는 중…")
- **오류 메시지 톤**: **친근하되 명확** (예: "앗, 자막이 없는 영상이에요. 다른 영상을 골라볼까요?")
- **사용자 제어**: 모든 오류에는 **재시도** 또는 **이전 단계로 복귀** 옵션을 제공

## 7. 기술적 제약

- **지원 환경**: **로컬 실행 웹 대시보드** (브라우저 UI + 로컬 백엔드 API). macOS / Windows / Linux에서 `npm run dev` 및 백엔드 서버 기동으로 사용. 인터넷 연결 필수 (YouTube 접근 및 분석 서비스 호출).
  - v0.1까지는 CLI/데스크탑으로 기술되었으나, UI 요구사항(React + Tailwind 대시보드)이 확정되면서 v0.2에서 웹 대시보드로 조정됨.
- **성능 목표**
  - 검색 결과 5개 표시: 키워드 입력 후 **5초 이내**
  - 보고서 생성·저장 완료: 영상 선택 후 **3분 이내**
  - 전체 End-to-End(키워드 입력 → 보고서 파일 저장): **5분 이내**
- **가용성**: 단일 사용자 로컬 도구 기준 (별도 SLA 없음)
- **언어**: 입력(한국어 / 영어), 출력(v1은 한국어 고정)
